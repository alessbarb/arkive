# Private state workflow

Arkive is especially useful when a repository accumulates important local-only material.

## Typical examples

- local architecture notes
- internal drafts
- working `.env` files
- ignored experimental files
- repository-local research scratchpads

## Workflow

1. keep shared project history in Git
2. keep private local state out of commits
3. snapshot that local state deliberately with Arkive
4. restore only when needed

## Why this matters

Without a workflow like this, teams often end up with one of two bad patterns:

- sensitive local state gets committed by mistake
- valuable local state disappears because “it was not in Git anyway”

Arkive exists to give that state a better home.
